## **🔁 로직 순서**

1. 기능 입력 받기(1번 - 페어 매칭, 2번 - 페어 조회, 3번 -  페어 초기화, Q - 종료)
2. 1번 페어 매칭을 선택한 경우,
   - 과정, 레벨, 미션 입력 받기
   - 기존에 해당조합으로 맺어진 페어가 있는지 확인
   - 있다면 다시 매칭할거냐고 물어보기
       - 할거면 새로 매칭
       - 없다면 다시 과정, 미션, 입력 받기
   - 없다면 새로 매칭
3. 2번 페어 조회를 선택한 경우,
   - 과정, 레벨, 미션 입력 받기
   - 해당 조합으로 있는지 조회하기
     - 있으면 그거 출력
     - 없으면 에러 메시지
4. 3번 페어 초기화를 선택한 경우,
  - 모든 페어 매칭 정보 삭제

### **⚠️ 예외**

> 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`를 발생시키고, `[ERROR]`로 시작하는 에러 메시지를 출력 후 해당 부분부터 다시 입력을 받는다
>
1. 기능 입력시
   - 정해진 형식에 맞지 않은 경우
2. 과정, 레벨, 미션, 입력시
   - 형식 오류(ex) 백엔드, 레벨1, 자동차경주)
   - 없는 과정, 레벨, 미션인 경우
3. 이미 해당 매칭이 존재하는 경우, 다시 매칭할거냐고 물어봤을 때, 그 형식이 오류(네 | 아니오) 
3. 페어 매칭시, 매칭을 할 수 있는 경우의 수가 없는 경우
4. 페어 조회시, 해당 조합의 페어가 없는 경우
